question: go run と go build の違いを教えてください
answer: `go run` と `go build` は、Goプログラムを実行およびビルドするためのコマンドですが、それぞれ以下のような違いがあります。

1. 目的:
   - go run: プログラムを実行するためのコマンドです。コードを試すときや開発中に使うことが多いです。
   - go build: プログラムをビルドして実行可能ファイル（バイナリファイル）を生成するためのコマンドです。デプロイや配布を目的として使われることが多いです。

2. 処理内容:
   - go run: ソースコードをコンパイルし、一時的な実行ファイルを生成して実行します。実行が終わると、一時的な実行ファイルは削除されます。
   - go build: ソースコードをコンパイルし、カレントディレクトリや指定されたディレクトリに実行ファイルを生成します。生成された実行ファイルは手動で削除する必要があります。

3. 実行速度:
   - go run: コンパイルした後に実行するため、実行速度は比較的遅いです。
   - go build: 事前にコンパイルされているので、実行速度は高速です。
